# Python 后端迁移执行卡

> 用途：直接发给 Claude Code 的任务卡。  
> 原则：一次只做一个批次，做完停下，等确认后再进下一批。  
> 执行要求：优先使用 `superpowers:executing-plans`，严格按验收点推进。

## 通用要求

- 不要同时改多个批次的内容
- 不要提前重构未进入批次的模块
- 保持 `/api/v1` 不变
- 保持 Web 端和 Flutter 端的调用契约尽量不变
- 每批必须输出：
  - 改了哪些文件
  - 跑了哪些验证
  - 还有什么风险

---

## 批次 1：Python 后端骨架

### 目标
把 Python 后端先跑起来，确认健康检查和路由前缀正确。

### 任务范围
- 建 FastAPI 项目骨架
- 加基础配置
- 加基础路由
- 加健康检查
- 保持端口和现有前端代理兼容

### 验收标准
- `GET /api/v1/health` 返回成功
- 后端可独立启动
- 测试可运行

### 输出要求
- 说明项目入口在哪里
- 说明如何启动
- 说明路由前缀如何保持一致

## 发给 Claude Code 的使用方式

1. 每次只发一个批次
2. 明确要求它不要跨批次扩展
3. 做完后要求它只汇报结果，不继续往下做
4. 下一批开始前先确认上一批通过验收
