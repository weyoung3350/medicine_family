# 移动端需求与任务计划

> 适用范围：Flutter Android/iOS App  
> 不包含微信小程序。

> 验证优先级：iPad 优先，Android 只做代码兼容约束，不要求实机验收。

## 1. 移动端定位

- 作为家庭成员日常使用端
- 重点承接服药提醒、今日打卡、药箱查看、AI 问诊、病历查询
- 适合老人、子女、家庭主账号共同使用
- 与 Web 端分工明确：Web 偏管理，移动端偏日常操作

## 2. 当前已具备的能力

### 2.1 账号与首页
- 登录与会话保持
- 登录后进入主页
- 底部五栏导航：概览、服药、药箱、AI、更多

### 2.2 概览页
- 家庭切换
- 今日待服统计
- 临期药品提醒
- 低库存提醒
- 今日服药时间轴

### 2.3 服药模块
- 今日服药列表
- 服药计划列表
- 依从性统计
- 打卡、跳过操作

### 2.4 药箱模块
- 药品列表
- 药品详情
- 拍照 OCR 识别
- 手动新增药品
- 库存展示

### 2.5 AI 模块
- 文本问诊
- 图片分析
- 咨询历史
- 快捷提问入口

### 2.6 其他模块
- 家庭管理
- 健康档案
- 病历管理
- 附近药店
- 设置与退出登录

## 3. 当前主要缺口

### 3.1 适老化大字模式不够独立
- 现在只有常规页面的大字号风格
- 缺少真正的“老人专用模式”
- 缺少最少步骤的打卡入口
- 缺少高对比和低干扰的专属布局

### 3.2 站内消息中心缺失
- 服药提醒、临期提醒、低库存提醒没有统一入口
- 缺少消息列表、未读数、已读状态
- 缺少对提醒消息的集中处理

### 3.3 服药计划创建流程还不够顺手
- 现在能创建计划，但入口和引导不够明确
- 缺少从 OCR 结果直接生成计划的流程
- 缺少处方或病历识别后的“一键转服药计划”

### 3.4 AI 返回展示还偏文本化
- 结果主要是聊天气泡
- 缺少结构化卡片
- 缺少风险提示、建议摘要、关键点分组

### 3.5 表单和状态同步还可以更稳
- 家庭切换、成员切换、计划切换需要更清楚
- 一些空状态和错误状态还偏简单
- 重新加载时的反馈不够统一

### 3.6 药箱录入链路还不够完整
- OCR 后可以手动修正，但还没形成完整确认流
- 库存新增与药品详情的引导还可以更直接
- 批次、效期、剩余量的展示还不够集中

## 4. 需求整理

### P0 核心需求
1. 适老化大字模式
2. 今日服药一键打卡
3. 家庭切换与成员切换稳定可用
4. 药箱基本查看、拍照识别、手动录入

### P1 重要需求
1. 站内消息中心
2. 服药计划 OCR 辅助创建
3. AI 结构化结果展示
4. 病历和药品之间的关联入口

### P2 增强需求
1. 更完整的空状态、错误状态、加载状态
2. 更清晰的药箱详情与库存操作
3. 更好的个人设置和家庭管理体验

## 5. 任务计划

### 批次 1：适老化模式
目标：
- 做一个独立的老人使用入口

任务：
- 新增老人模式首页或简化页
- 增大按钮、字号、间距
- 减少视觉干扰
- 强化“今日服药”主入口

验收：
- 老人模式能直接看到今日服药
- 打卡操作步骤减少
- 页面在小屏上不溢出

### 批次 2：站内消息中心
目标：
- 把提醒统一到一个入口

任务：
- 消息列表页
- 未读数
- 已读/全部已读
- 从首页或更多页进入

验收：
- 能看到提醒消息
- 能标记已读
- 消息和服药/库存/临期关联清晰

### 批次 3：服药计划 OCR 辅助
目标：
- 从病历或处方图片辅助创建计划

任务：
- 识别结果页
- 结果确认页
- 一键生成计划草稿
- 计划前置预览

验收：
- OCR 结果能转成计划草稿
- 用户能修改后再提交

### 批次 4：AI 结构化展示
目标：
- 让 AI 结果更像“健康建议卡片”，不是纯聊天记录

任务：
- 结构化结果渲染
- 风险提示卡
- 建议摘要卡
- 关键点折叠区

验收：
- AI 结果可分段展示
- 风险和建议能快速识别

### 批次 5：药箱和家庭体验收口
目标：
- 让日常操作更顺手

任务：
- 药箱详情优化
- 库存批次展示
- 家庭/成员切换优化
- 空状态与错误状态统一

验收：
- 切换成员和家庭不混乱
- 药箱信息更清晰
- 页面状态一致

## 6. 推荐实现顺序

1. 适老化模式
2. 站内消息中心
3. OCR 辅助计划
4. AI 结构化展示
5. 药箱和家庭体验收口

## 7. 给 Claude Code 的执行要求

- 每次只做一个批次
- 不要同时改 Web 和移动端
- 不要提前碰下一批功能
- 先写测试或最小验证，再改页面
- 每批结束后说明：
  - 改了哪些文件
  - 主要交互变化
  - 还有哪些未完成项
