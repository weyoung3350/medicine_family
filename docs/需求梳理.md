# 家庭健康管家需求梳理

## 1. 项目目标
- 面向家庭的健康管家
- 集中管理家庭成员健康信息、药品库存、服药计划、病历资料和 AI 问诊
- 形成“录入 - 管理 - 提醒 - 分析”的使用闭环
- 面向老人等被代管成员提供可独立使用的适老化操作体验

## 2. 角色与对象
- 家庭主账号：负责创建家庭、管理成员、查看整体状态
- 家庭成员：使用服药提醒、查看健康信息、参与问诊
- 被代管成员：如老人、儿童或需要长期照护的家庭成员

## 3. 核心功能

### 3.1 账号与家庭管理
- 用户注册、登录、退出
- 创建家庭、加入家庭
- 邀请码加入家庭
- 家庭成员管理
- 成员健康档案维护

### 3.2 健康档案
- 维护成员基础信息
- 记录成员关系
- 记录既往病史、过敏史、长期用药等信息
- 健康档案可供 AI 问诊和用药判断参考

### 3.3 药箱与库存管理
- 药品手动录入
- 药品 OCR 识别录入
- 药品基础信息管理
- 库存批次管理
- 剩余量追踪
- 临期预警
- 低库存预警

### 3.4 服药计划与打卡
- 创建服药计划
- 支持不同频次
- 支持多个服药时间点
- 支持 OCR 识别生成服药计划
- 服药打卡、跳过、漏服记录
- 打卡后自动扣减库存
- 服药依从性统计

### 3.5 适老化大字打卡模式
- 提供适合老人直接操作的首页或打卡入口
- 提供大字、高对比度、低干扰的界面
- 提供一步完成的服药确认流程
- 提供简化信息展示，降低误操作概率
- 可结合震动、语音或明显状态反馈提升可感知性

### 3.6 AI 健康助手
- 日常问答
- 药物相互作用检查
- 服药指南生成
- 诊疗单、病历、化验单图片分析
- 结合健康档案、库存、当前用药给出建议

### 3.7 病历管理
- 病历录入与查看
- 病历 OCR 识别
- 支持按成员、关键词筛选
- 病历可关联处方药品信息

### 3.8 附近药店
- 基于定位搜索附近药店
- 支持距离范围筛选
- 支持关键词搜索
- 地图与列表展示

### 3.9 通知与定时任务
- 服药提醒
- 漏服提醒
- 临期提醒
- 低库存提醒
- 以站内信方式向家庭管理者或相关成员发送提醒
- 三方推送、短信、邮件等能力可作为后续扩展

## 4. 非功能要求
- API 统一前缀
- 登录鉴权统一
- Web 与移动端尽量保持一致的数据结构
- AI 结果需要可追溯
- 关键识别结果应支持人工确认

## 5. 需求边界
- AI 结果用于辅助决策，不替代医生诊断
- OCR 识别结果不应直接作为最终数据，需人工校对
- 通知能力属于业务闭环的一部分，不能只保留日志输出

## 6. 实施优先级建议
1. 家庭、健康档案、药箱、服药计划主链路
2. 适老化大字打卡模式
3. OCR 录入与人工确认
4. AI 问答与病历分析
5. 站内信提醒与消息中心
6. 附近药店与扩展能力

## 7. 现阶段建议补强
- 成员“关系”字段标准化，避免同义词混乱
- 适老化模式单独设计，不要只依赖现有页面放大
- 服药计划 OCR 增加人工确认步骤
- 临期/低库存预警先用站内信闭环，三方推送后置
- AI 问诊增加更明确的业务约束和结构化输出
- 补充接口文档和数据字典，方便联调和验收
